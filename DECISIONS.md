# Decisions

更新日期：2026-07-15

## 1. 新建统一项目，不继续拼旧项目

决定：以 `crypto-monitor` 为统一项目，旧的 KDJ 告警和 Trump 监控只作为迁移来源。

原因：需要统一后台配置、数据库、通知、worker 生命周期和前端布局。

## 2. `.env` 只放启动级配置

决定：业务配置放 SQLite 和后台管理，`.env` 只保留启动级配置。

原因：用户需要在线修改策略、机器人、翻译、清理和巨鲸配置。

## 3. `RUN_WORKERS=true` 是生产默认

决定：生产环境默认开启 `RUN_WORKERS=true`。

原因：服务启动后应自动轮询和推送，不依赖前端网页打开。

影响：关闭后 API 可用，但策略、新闻、通知、清理、巨鲸 worker 不运行。

## 4. 使用 systemd，而不是 nohup

决定：服务器后台常驻使用 systemd。

原因：systemd 支持开机自启、状态查看、日志、崩溃重启，比 `nohup` 更适合长期服务。

## 5. 直接用 Uvicorn 暴露 8800，暂不做 HTTPS

决定：当前访问 `http://167.179.69.248:8800/`。

原因：还没有域名和 TLS 反代。

注意：`https://167.179.69.248:8800/` 不会正常工作。

## 6. 敏感字段加密保存

决定：Webhook URL、bot token、翻译 API key、DeBank key 等用 `APP_SECRET_KEY` 加密。

原因：后台和数据库会保存真实密钥，不能明文暴露给前端。

## 7. 前台展示周期和机器人推送周期分离

决定：

- 前台周期只影响用户当前看什么。
- 后台策略周期决定机器人推送什么。

原因：用户反复强调二者不是一回事。

## 8. KDJ/MA/BOLL 属于行情与策略监控内部

决定：不把 KDJ、MA、BOLL 恢复成独立模块。

原因：用户希望每个币下方按策略分组展示，不要模块列表混乱。

## 9. 机器人模板和前台短文案分离

决定：

- 前台卡片使用短文案，一行尽量可扫读。
- 机器人使用详细模板。

原因：看板空间有限，机器人需要完整上下文。

## 10. 机器人技术告警移除 K 线时间

决定：所有机器人提醒不再显示“K线时间”。

原因：用户明确要求去掉该字段，避免和提醒时间混淆。

## 11. BOLL 机器人顺序为上、中、下

决定：BOLL 模板字段顺序固定为上轨、中轨、下轨。

原因：用户明确要求。

## 12. 清理策略默认开启

决定：新增 `cleanup` 策略，默认每天北京时间 12:30 执行。

默认保留：

- 告警 30 天
- 新闻 60 天
- 巨鲸事件 90 天

原因：服务器容量有限，容量优先。

## 13. 清理不做本地备份

决定：删除超期数据后直接 `VACUUM`，不生成本地备份。

原因：本地备份会让服务器容量翻倍占用。

## 14. 社媒媒体帖必须补详情页

决定：RSS 只给空正文、链接或 HTML 时，对疑似媒体/转发帖抓详情页。

原因：特朗普经常发图片、视频、转发、纯链接内容；不能把 HTML 残片当正文。

## 15. Truth Social 跳转统一到 detail URL

决定：特朗普原帖优先跳 `https://truthsocial.com/@realDonaldTrump/posts/{id}`。

原因：用户明确要求，不要跳中间 RSS 页面或错误卡片链接。

## 16. 翻译错误 token 不显示

决定：前端过滤“请提供英文新闻标题或摘要”等翻译失败提示。

原因：这些是模型错误解释，不是新闻内容。

## 17. 转发卡片也要翻译

决定：转发卡片的 `description` 翻译写入 `metadata.card.translated_description`。

原因：转发正文常在卡片里，不在主正文里；前台应显示中文。

## 18. 巨鲸第一版只做只读监控

决定：不做跟单、下单、签名、钱包授权。

原因：风险高，且用户明确不需要跟单按钮。

## 19. 巨鲸主源先用 Hyperliquid

决定：合约持仓和成交监控优先接 Hyperliquid 公共 Info endpoint。

原因：Hyperliquid 可公开查询用户持仓、成交、委托、资金流水，低成本且适合截图需求。

## 20. DeBank 是可选多链资产源

决定：DeBank AccessKey 未配置时，现货/DeFi 显示未配置，不影响 Hyperliquid。

原因：DeBank 适合多链资产和协议仓位，但生产使用通常需要 key。

## 21. 聪明钱标签先手动

决定：聪明钱、巨鲸、KOL、机构、做市商、交易员、重点关注等作为本地标签选择。

原因：实体归因不是链上原生事实，低成本版先由用户确认。

## 22. 巨鲸推送规则：有新操作就提醒

决定：关注地址有新 Hyperliquid 成交操作就机器人提醒。

原因：用户明确说后台阈值是给前台看板看的，不作为机器人推送门槛。

## 23. 巨鲸阈值只用于前台“大额”标记

决定：成交额阈值和币种数量阈值只影响前台标记。

原因：避免漏掉关注对象的实际操作。

## 24. 首次巨鲸同步不补发历史通知

决定：首次同步建立游标和展示数据，不发历史机器人消息。

原因：避免上线时刷屏。

## 25. Hyperliquid 使用保守轮询频率

决定：

- 主轮询不低于 300 秒。
- 成交轮询不低于 120 秒。
- 扩展信息不低于 1800 秒。

原因：Hyperliquid 偶发 SSL EOF，不一定是限频，但需要减少压力和误报。

## 26. 巨鲸机器人模板去掉地址和链接

决定：成交提醒模板不显示地址、链接、大额标记。

原因：用户明确要求模板更短，只保留操作核心信息。

## 27. Open Long / Close Short 必须中文化

决定：前台和机器人不直接显示 Hyperliquid 英文动作。

映射：

- Open Long：买入开多
- Close Long：卖出平多
- Open Short：卖出开空
- Close Short：买入平空

## 28. UI 偏好要终端隔离

决定：主题、布局、周期、图表指标等用户偏好应保存在本地，不应互相影响。

原因：多个终端同时访问时，一个终端切换主题不应影响另一个终端。

## 29. 后台策略配置分组

决定：后台策略配置按类别分组：

1. KDJ / MA / BOLL
2. 巨鲸
3. 社媒和新闻
4. 翻译和清理

原因：原来全部混在一起难以管理。

## 30. 巨鲸关注对象管理放到巨鲸模块

决定：关注对象 CRUD 从后台策略配置挪到巨鲸模块。

原因：关注对象是业务数据，不是策略参数；用户在看巨鲸时应直接管理。

## 31. 图表刷新不能清空 DOM

决定：lightweight-charts 销毁时只调用 `chart.remove()`，不再 `container.innerHTML = ""`。

原因：React StrictMode 下会触发 `removeChild` crash，导致本地黑屏。

## 32. 前端 `index.html` no-cache

决定：后端返回 `index.html` 时加 no-cache headers。

原因：生产环境刷新可能拿到旧 HTML 指向不存在的旧 asset hash。

## 33. ETH 大图默认显示 MA

决定：ETH 大图默认展示 MA7、MA25、MA99，其他指标默认不显示。

原因：用户要求默认显示 MA 三条，其他指标点击切换显示，不能同时显示。

## 34. 技术策略监控范围和机器人提醒币种分离

决定：KDJ、MA、BOLL、BOLL中轨/MA 使用 `symbols` 保持前台监控范围，使用独立的 `notify_symbols` 控制机器人提醒币种；空列表表示不推送。

原因：用户需要继续在前台查看全部启用币种的策略信号，但飞书只接收选中的一两个币种。

## 35. BOLL中轨/MA交叉作为独立策略并归入BOLL前台区域

决定：新增 `boll_ma_cross` 独立配置、独立机器人绑定，默认 BOLL(20)/MA(99)；前台不增加第四栏，而是合并到现有 BOLL 告警区域。

原因：后台需要独立管理参数、周期、币种和机器人，用户界面仍保持 KDJ/MA/BOLL 三组结构。

## 36. 技术指标推送采用“指标 × 币种 × 周期”矩阵

决定：四个技术策略统一监控全局启用币种，并分别使用 `notify_intervals_by_symbol` 作为唯一推送规则；后台按策略标签页集中管理矩阵和该策略绑定的机器人。旧 `symbols`、`notify_symbols` 与 `intervals` 保留用于升级兼容，但不再由新界面编辑。

原因：币种集合和周期集合做笛卡尔积会产生用户未选择的组合，无法表达“BTC/KDJ/5m、BTC/BOLL/1h、ETH/MA/4h”这类精确订阅。矩阵仍只影响机器人推送，不缩小前台监控范围。
