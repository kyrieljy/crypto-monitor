# 决策记录

## 1. 新建集成项目，不直接复用旧项目为主项目

决定：

- 新建 `market-monitor-dashboard`。
- 旧项目 `crypto-kdj-alert` 和 `trump-monitor` 只作为迁移来源。

原因：

- 新项目需要统一后台管理、看板布局、数据库、通知和 worker 生命周期。
- 直接拼接旧项目会导致配置分散、策略和通知无法统一管理。

## 2. `.env` 只保留启动级配置

决定：

- 业务配置迁移到后台管理和 SQLite。
- `.env` 只保留数据库路径、端口、管理员密码、密钥、worker 开关、超时等启动级配置。

原因：

- 用户需要即时调整 KDJ、MA、BOLL 参数和通知周期。
- Webhook、新闻源、翻译配置需要在后台管理。

## 3. 敏感字段加密保存，读取时脱敏

决定：

- Webhook URL、bot token、chat id、翻译 API key、巨鲸 API key 等敏感字段使用 `APP_SECRET_KEY` 加密。
- API 返回 masked 值。

原因：

- 后台会管理真实 webhook/token。
- 避免前端和接口直接泄露完整密钥。

## 4. 后台必须先登录才能看内容

决定：

- 点击后台进入登录页。
- 未登录不展示任何配置内容。
- 登录后显示配置，支持登出。

原因：

- 用户明确要求从“未登录可查看、只限制修改”改为“登录后才能看内容”。
- 后台包含 webhook/token 相关敏感配置。

## 5. 首页展示周期和机器人提醒周期分离

决定：

- 前台看板的 K 线周期和策略展示周期只影响用户当前看什么。
- 后台周期多选框控制机器人提醒哪些周期。

原因：

- 用户明确说明：前台所有周期都监控，用户选择看；后台配置机器人提醒什么周期。
- 如果混在一起，会让用户误以为看板选择会改变真实推送策略。

## 6. KDJ、MA、BOLL 属于行情与策略监控内部分类

决定：

- 后台模块显示不再有独立的 `KDJ 模块`、`MA 模块`、`BOLL 模块`。
- 三个策略在 `行情与策略监控` 内按币种和策略分类展示。

原因：

- 用户认为独立模块没用，要求删除。
- 首页顶部五币下面按 KDJ/MA/BOLL 分类更清楚。

## 7. 行情与策略监控不拖拽

决定：

- 第一行行情与策略监控固定宽度和位置。
- 不再使用内部横向滚动条。
- 不跟其他模块一起拖拽。

原因：

- 这是核心看板，用户要求默认能看完所有策略。
- 拖拽和内部滚动影响读取效率。

## 8. 默认看板布局四行

决定：

- 第一行：行情与策略监控。
- 第二行：巨鲸与聪明钱动态。
- 第三行：特朗普社媒监控、白宫发言新闻。
- 第四行：最近告警、数据源健康。

原因：

- 符合用户对信息优先级的要求。
- 巨鲸和聪明钱不应被放在第三行以下或混入策略图。

## 9. 前台策略文案和机器人推送模板分离

决定：

- 前台策略卡片使用短文案，例如 `5m J下穿K，收盘价667.54`。
- 机器人使用详细模板，含标的、周期、信号、指标值、K 线时间、数据源、提醒时间。

原因：

- 前台卡片宽度有限，需要短、可扫读。
- 机器人推送需要完整上下文。
- 用户明确给出 MA、KDJ、特朗普社媒模板，并要求 BOLL 和白宫举一反三。

## 10. BOLL 使用收盘价突破

决定：

- 默认 `period=20`、`stddev=2`。
- 当前 K 线收盘价上穿上轨或下穿下轨才告警。
- 默认按闭合 K 线，可配置实时 K 线。
- 去重按币种 + 周期 + 方向 + K 线时间。

原因：

- 用户明确要求 1h 和 4h 只要上穿上轨或下穿下轨就报警。
- 收盘价判断比 intrabar 突破更稳定。

## 11. Truth 社媒默认 RSS 优先

决定：

- `TrumpTruth RSS` 默认开启。
- `Truthbrush` 保留可选，但默认关闭。

原因：

- 用户担心 Truthbrush 和 RSS 同时开会重复。
- 当前 RSS 路径足够做 MVP。

## 12. White House 需要关键词配置

决定：

- White House 后台支持 include keywords 和 exclude keywords。

原因：

- 用户问白宫发言是否有关键词、是否需要配置。
- 关键词可以控制噪声和漏报风险。

## 13. 大模型翻译不是 webhook

决定：

- 翻译策略只配置 API 地址、模型名、API key、温度。
- 模型当前应使用 `deepseek-v4-flash`。
- 翻译只处理社媒和新闻文本。

原因：

- 用户指出“大模型翻译为什么可以选择 webhook”是不合理的。
- webhook 是通知目标，不是翻译服务。

## 14. 巨鲸功能不能伪造

决定：

- 没有巨鲸 API 文档前，只做 provider adapter、关注对象、事件表、后台配置、UI 占位。
- 不写死假接口当真实能力。

原因：

- 用户明确说后续要通过巨鲸监控工具 API。
- 假数据只能用于 UI 测试，必须和真实能力区分。

## 15. 数据源灾备要在前台可见

决定：

- 当行情源使用灾备时，在行情模块顶部显示当前生效源。

原因：

- 用户关心美国网络下币安 futures 是否可用、是否应该切换数据源。
- 灾备状态需要直接暴露，方便判断数据可靠性。

## 16. 后台提示使用居中弹框

决定：

- 成功/失败提示不放账号区域。
- 弹框页面居中。
- 关闭按钮放右上角。
- 报错尽量解析成精确中文原因。

原因：

- 用户明确要求。
- 后台保存、测试、登录等操作需要明确反馈。

## 17. 开关只能点击开关本体触发

决定：

- `Switch` 使用独立 button，不用 label 包裹整行。
- 模块显示行不再用 label 触发 checkbox。

原因：

- 用户发现有些开关点同行其他位置也会触发。
- 误触开关会导致后台配置混乱。

## 18. 侧边栏收起后主内容保持原宽度居中

决定：

- 侧边栏收起后不让主内容无限扩张。
- 主内容保持接近展开时宽度，并在右侧区域居中。

原因：

- 用户认为收起后内容太宽太满。
- 交易看板需要留白，避免密度过高影响阅读。
